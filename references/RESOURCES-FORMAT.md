# RESOURCES.md Format

Ported from `/teach`, unchanged in spirit and stricter in practice: this workspace's output gets published, so the "never trust parametric knowledge" discipline is a hard floor, not a nice-to-have.

## Structure

```md
# {Topic} Resources

## Knowledge

- [Article: "..." — Author (Publication)](https://example.com)
  One line: what it covers, when to reach for it.

## Wisdom (Communities)

- [r/subreddit](https://reddit.com/r/subreddit)
  One line: what it's good for, moderation quality.

## Gaps

- {Area the mission needs where no good source has been found yet}
```

## Rules

- **High-trust only.** Primary sources, recognised experts, peer-reviewed or well-moderated communities. Marketing dressed as education doesn't belong.
- **Annotate every entry** — a bare link is dead weight in three months.
- **Every factual claim in a module traces back to something here — and this doesn't end at Explore Mode.** A Create Mode artifact (video script, blog post, essay, presentation) is still teaching someone, just a different audience than the explorer — so the same rule applies to drafting an output artifact as to writing a module: nothing gets introduced during drafting that wasn't already grounded in a module, a confirmed highlight, or an entry here. If a module — or a draft artifact — is about to state something and there's no source in `RESOURCES.md` to back it, go find one before writing the claim, don't write it from memory and cite it after the fact.
- **Surface gaps explicitly** in a `## Gaps` section — drives what to search for next.
- **Prune ruthlessly.** A source that turned out shallow or wrong gets removed, not buried.
