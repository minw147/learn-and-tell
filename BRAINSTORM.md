## 2. 核心差异化：Problem Posing（让学习者提出问题）

### 市场现状（99% 的 AI 学习工具）
```
AI 出题 → 用户做题 → AI 批改 → 下一题
```
这测试的是 **recall（回忆）**，只能反映 fluency（即时流畅度），不反映 storage strength（深层理解）。

### 本项目的反转设计
```
AI 教概念 → 用户自己提出一个问题/场景 → AI 评估问题的质量 → 自适应调整
```

**"评估你提出的问题"比"评估你回答的问题"难得多，也更能反映真实理解。**

这直接来自 Min 的博士研究方向 —— **Problem Posing**。不是硬造的差异化，是从学术根基里长出来的护城河。

> 💡 一个能提出好问题的人，一定比一个能答好题的人理解得更深。

## 5. 用户流程（初步构想）

```
1. 选主题（数学 / 历史 / AI / 自定义输入）
        ↓
2. 简短的 mission 确认（你为什么想学这个？）
   → 用于 ground 后续教学，Min 的 "why learner" 方法论
        ↓
3. 第一节 mini-lesson（自适应难度，基于 mission）
   - 核心概念（类比驱动，step-by-step）
   - 一个 tangible win（小成就感）
        ↓
4. ⭐ Problem Posing（关键差异化交互）
   "基于刚才学的，你能不能提出一个相关问题或场景？"
   用户输入 → AI 分析：
   - 概念准确性（理解对了吗？）
   - 深度（表层复述 vs 深层应用）
   - 创造性（照搬 vs 举一反三）
        ↓
5. AI 反馈 + 自适应下一轮难度
        ↓
6. 学习进度可视化（掌握度 / 知识点地图）
```

### 第4步是 Wow Moment
面试官 / 用户看到"AI 评估你提出的问题"会觉得：这个人不只是在用 AI，她在用**认知科学**重新设计学习。

### ⭐⭐⭐ 核心 IP：Content Complexity Scoring Rubric（0-5 评分体系）

这是 Min 在 Math Walks 论文中验证过的、可以直接转化为 AI prompt 的评估标准：

**三个评估维度：**
1. **Relevance（相关性）**：问题是否与 prompt/情境相关
2. **Ambiguity（歧义性）**：问题陈述是否清晰无歧义
3. **Multiple Solutions（多解性）**：问题是否允许多种解法/视角

**0-5 分评分标准（已验证，Cohen's kappa = 0.81，good agreement）：**

| 分数 | 类别 | 含义 | 示例 |
|------|------|------|------|
| **0** | Not-relevant / incomprehensible | 无关或无法理解 | "All circles together." |
| **1** | Relevant statement | 相关陈述，但不是问题 | "This could be a probability question." |
| **2** | Relevant problem, with ambiguity | 相关问题但有歧义 | "Why were they built like that?" |
| **3** | Relevant problem, no ambiguity | 清晰的相关问题 | "From just looking at the picture, how many circles can be calculated by each color?" |
| **4** | Non-routine, no ambiguity | 非常规问题，清晰无歧义 | "If the real estate agency wanted to renovate and deduct 10 meters in the living room... what would be the area?" |
| **5** | Non-routine, no ambiguity, multiple solutions | 非常规 + 清晰 + 多解 | "How do the color and space between each color make this picture pleasing to the eye?" |

> 💡 **这就是产品可以直接用的评估引擎。** AI 拿到用户提出的问题，按这三个维度打 0-5 分，给出具体反馈。不是拍脑袋的评估，是有学术验证的标准。

### Min 自己总结的三个关键洞察（对话中提炼）

1. **"费老师"是最大 blocker**
   - Problem-posing 是极好的学习方法，但"非常费老师"
   - 每个学生提出的问题都是独一无二的，没有标准答案能快速评判
   - **AI 恰好解决了这个 blocker** —— 这就是产品的核心价值主张

2. **提升 metacognitive skill（元认知能力）**
   - 很多学生欠缺元认知 —— 不知道自己知不知道
   - Problem-posing 强制学生反思"我到底理解了没有"
   - 产品可以显性化这个元认知过程

3. **知识框架可视化**
   - 学习是把新知识放进已有框架
   - 可视化的思维导图/知识架构对学生帮助极大
   - "根据你提出的问题来看你对这个事情的了解程度" —— 问题质量是理解度的窗口

### Math Walks 概念的产品化潜力

Min 的 "Math Walks" 研究（在社区/真实世界场景中做 problem-posing）有一个有趣的数字化延伸方向：
- 用户看到一张图/一个场景 → 基于这个场景提出数学问题 → AI 评估
- 这可以变成移动端体验（拍照 → 提问 → 评估）
- 但 MVP 阶段先用文本/图片 prompt 即可

## 10. ⚡ 关键转折：从"评分引擎"到"对话驱动器"

> 2026-07-11 头脑风暴中的方向调整 —— 这可能是整个产品最重要的设计决策

### Min 的核心洞察

**"问题没有错误的问题。"**

回答一个数学题或历史问题，有对错。但问题本身没有"错误的问题" —— 只有相对更好、更有趣的问题。这个哲学差异是 problem-posing 不同于传统测试的根本原因。

由此引出一个重要质疑：

> **我们到底要不要"打分"用户提出的问题？还是说，问题的真正价值是作为学习对话的燃料？**

### 两种产品哲学的对比

| | A: 评分引擎（原方案） | B: 对话驱动器（新方向） |
|---|---|---|
| **核心动作** | 用户提问 → AI 打 0-5 分 | 用户提问 → AI 用问题驱动下一轮学习 |
| **用户的感受** | "我的问题好不好？"（可能焦虑） | "我的问题引导了下一步学什么"（赋能） |
| **评判标准** | 复杂度、深度（需要 rubric） | 这个问题是否揭示了理解状态 / 是否有探索方向 |
| **无问题时** | 得 0 分（负面） | 信号：这课太简单了 → 进入下一话题（正面/中性）|
| **哲学** | 形成性 vs 总结性的张力 | 纯形成性（formative），无评判 |

### ⭐ Min 提出的"双向问答"模型

这是她真正想要的交互设计 —— 把 Teach skill 已有的 quiz 模块**扩展成双向**：

```
每节课结束后：
  ┌─ AI 问你（已有）：验证你学到了什么
  │   "用今天学的概念解释 X"
  │
  └─ 你问 AI（新增）：揭示你想到了什么
      "这个我之前听过，但不确定是不是我想的那样"
      "这两个概念之间有联系吗？"
      "这个跟我现在的课有什么关系？"
      "下一节课我想学 ___"
```

**用户提出的问题本身就是理解度的信号：**
- 问的是表层复述 → 说明只到 fluency
- 问的是跨概念连接 → 说明在建立 storage strength
- 问的是"我之前听过但不确定" → 说明旧知识被激活了
- 问的是"下一节我想学什么" → 说明在主动建构学习路径
- **什么都不问** → 可能太简单了，该往前走了

### 这为什么是更好的方向

1. **消除了"我的问题够好吗"的焦虑** —— 没有"打分"，问题就是对话
2. **契合 metacognitive skill 的本质** —— 元认知不是被测出来的，是在对话中涌现的
3. **"无问题"从负面变成正面信号** —— 自然的自适应逻辑（没疑问 = 该升级了）
4. **更接近真实学习体验** —— 这就是 Min 用 Teach skill 学习时的真实体验
5. **更难被竞品复制** —— 竞品在做"AI 批改你的答案"，我们在做"AI 跟你一起思考"

### 学术角度的思考

Problem-posing 研究里确实存在**形成性（formative）vs 总结性（summative）**的张力：
- 总结性：用 problem-posing 来评估学生水平（需要 rubric 打分）
- 形成性：用 problem-posing 作为教学手段本身（Min 现在倾向的方向）

Min 自己的研究（Math Walks）其实两种都做了 —— 她用 0-5 复杂度做了量化评估（总结性），同时质性分析发现学生"开始更深入地思考、提问、连接现实场景"（形成性效果）。

**产品可能不需要二选一：** 可以用复杂度分析在后台驱动自适应（用户看不到分数），而前台只呈现对话式的探索。

## 11. ⭐ 核心理论锚点：Metacognitive Mirror（元认知镜像）

> 2026-07-11 —— Min 将 problem-posing、双语认知研究、AI agent loop 三者打通

### 三个领域的连接

Min 指出了一个深层共性 —— 以下三件事本质上是同一个认知过程：

| 领域 | 这个过程叫什么 | 怎么运作 |
|------|--------------|---------|
| **Problem-posing 教育** | Metacognitive skill | 学生提出问题 → 反思自己的理解状态 |
| **双语认知研究** | Metalinguistic awareness / 执行功能 | 双语者反复在两种语言间切换 → 需要监控"我现在在用哪个系统" → 元认知能力更强（有文献支持）|
| **AI coding agent** | Self-verification loop | 生成代码 → 自我评估 → 不可行则调整 → 再试 |

**核心共性：** 生成一个东西 → 在脑海里自我评估它 → 根据评估调整。这个循环本身就是高阶认知能力的本质。

### 产品的理论定位（基于此洞察）

> **这个产品不是"教你知识"的，是帮你练就"能自我监控、自我校准"的元认知肌肉。**

Problem-posing 是训练手段，不是目的。目的是让用户发展出"我自己能评估自己的想法好不好"的能力 —— 就像一个 coding agent 不断在 loop 里自我验证。

### 为什么"不评判"是对的

如果 AI 打分（"你的问题得 3 分"），就把这个训练机会**外包**给了 AI —— 用户不再需要自己评估，AI 替他评估了。

**不打分反而更有效，因为：**
- 用户必须自己面对"我这个问题到底说明我懂了还是没懂"
- 没有外部裁判 → 被迫启动内部裁判（metacognitive process）
- AI 的角色是**镜像**（mirror），不是**裁判**（judge）—— 它帮你看到自己的想法，但不替你评判

### 双语学习者这个角度的独特价值

Min 提到的双语 → 元认知增强的文献，是一个潜在的差异化叙事：
- 会多语言的人，元认知/执行功能天生更强（因为要频繁监控语言切换）
- 单语者这方面的锻炼机会少
- **这个产品本质上是在用 problem-posing 给单语者/所有人提供一种"类似双语体验"的元认知训练**

> 💡 这个叙事在 marketing 和 funding pitch 时会非常有力量 —— 不只是"学习工具"，而是"认知增强工具"。

### 对产品设计的影响

| 设计决策 | 导向 |
|---------|------|
| AI 不打分用户的问题 | ✅ 保持元认知训练不被外包 |
| AI 问"你觉得这个问题指向什么？" | ✅ 促发自我评估 |
| AI 复述/结构化用户的问题 | ✅ 镜像功能 —— 帮用户看清自己的想法 |
| AI 问"下一节你想学什么？" | ✅ 让用户主动校准学习路径 |
| AI 给"好/坏问题"的标签 | ❌ 破坏元认知训练，变成外部裁判 |

## 12. 跨领域 Problem-Posing 框架（文献速览）

> 数学方向 Min 是权威，已全部读过，不再重复检索。以下仅为人文社科方向的补充，用于理解跨领域差异。

### 历史教育：Inquiry Arc & Historical Reasoning

历史领域的 problem-posing 与数学完全不同：

| 维度 | 数学 | 历史 |
|------|------|------|
| 框架名称 | Complexity scoring（Min 的 0-5）| Inquiry Arc / Historical Reasoning（Van Drie 等）|
| 问题性质 | 基于场景构造可解的数学问题 | 质疑叙事、比较来源、分析权力关系 |
| 好问题标准 | 相关性 + 清晰度 + 多解性 | **问题能否引发深度探究** |
| 关键发现 | — | **评估性问题 > 解释性问题**更能激发历史思维 |

> 来源：Van Drie & van Boxtel, "Historical Reasoning: Towards a Framework" (Educational Psychology Review)；NYSED Social Studies Framework 的 Inquiry Arc（提出问题 → 收集证据 → 形成概括 → 论证）

### 创造力研究：Torrance 四维度

人文学科（创意写作、语言艺术）的"好问题/好作品"评估常用 **Torrance 创造性思维维度**：
- **Fluency（流畅性）**：能产生多少想法
- **Flexibility（灵活性）**：想法跨越多少类别
- **Originality（独创性）**：想法多罕见
- **Elaboration（详尽性）**：想法被发展得多充分

这套来自 TTCT（Torrance Tests of Creative Thinking），跟数学 complexity 是完全不同的评价哲学。

### ⭐ 跨领域的关键洞察

**不同领域定义"好问题"的方式不同，但"好问题驱动深度思考"是共性。**

- 数学的 complexity rubric 不适用于历史
- 历史的 inquiry arc 不适用于数学
- Torrance 的创造力维度又是另一套
- **但所有领域里，"问题作为探究/对话的燃料"都适用**

> 💡 这反过来**强化了"不打分"的设计决策** —— 因为没有一套通用 rubric 能跨领域评判问题质量。而"让问题驱动自适应学习"可以跨领域通用。产品的统一性不在评判标准上，而在交互模式上。

## 13. 🏆 竞品分析与差异化定位

> 2026-07-11 竞品调研

### 核心竞品 1：Khanmigo（Khan Academy AI Tutor）— ❌ 失败案例

**它做了什么：** Socratic 方法 —— 不给直接答案，用提问引导学生思考。

**为什么失败了：**

| 问题 | 细节 |
|------|------|
| **"5% 问题"** | Khanmigo 公布的正面结果中，**95% 的研究参与者被排除**才能得到"最强效果"。Dan Meyer 称之为 "5 Percent Problem"。Q-Chat 已死。 |
| **学生只想要答案** | 最大的用户投诉（尤其 K-5）：孩子在做题时想要帮助，结果得到的是更多问题。"When students just need a direct answer, being forced into a back-and-forth conversation wastes time and kills momentum." |
| **Socratic 方法是假的** | Punya Mishra 的批评："问问题 ≠ 苏格拉底式的提问"。真正的苏格拉底法是知道**问什么、何时问、为什么问**。Khanmigo 只是机械地"不给答案+反问"，学生只会反复回 "idk"。 |
| **不连接学生的世界** | Dan Meyer: "这对孩子来说是个无意义的问题，因为它根本不连接他们的世界。我们不能从无意义出发，用苏格拉底法走向意义。" |
| ** blamed 学生** | Khanmigo 的 DiCerbo 说"学生不擅长问问题" —— 但这被反驳为"任何认识小孩的人都知道孩子是最好奇的"。问题不在学生，在产品。 |

> **Min 的洞察（直接引用）：** "小孩做一个题，他不会去问这个 chatbot，他其实根本没学到东西。说是'不给你直接答案，引导思考'，但其实很难做到。"

**Khanmigo 失败的根本原因：** 它把**学生不想学的题**硬套上 Socratic 对话 —— 学生动机是"完成作业"，不是"理解"。在错误动机上叠加教学法，注定失败。

### 核心竞品 2：开源 AI Tutor 平台（DeepTutor / OpenTutor / Open TutorAI）

这几个都是 2025-2026 年涌现的开源项目：

| 项目 | 特点 | 与我们的差异 |
|------|------|------------|
| **DeepTutor** (HKUDS) | 终身个性化辅导，**Memory Graph**（L1 traces, L2 summaries, L3 synthesis），可自托管，社区 skills (EduHub) | 有长期记忆 + 可自托管 ✅ 但定位是"辅导"（帮学生理解材料），不是 problem-posing / 元认知训练 |
| **OpenTutor** (zijinz456) | Block-based 自适应学习 workspace，本地运行，上传材料 → AI 生成笔记/quiz/flashcard + 自适应 tutor，支持 10+ LLM | 有课件生成 + 本地运行 ✅ 但是"基于上传材料"，不是"学生自主发起的学习" |
| **Open TutorAI** | 开源教育平台，有 sessions/chat/个性化辅导支持，模块化架构 | 有 session 架构 ✅ 但偏传统"辅导"模式 |

**这些项目的共性 = 它们的机会窗口也是我们的：**
- 都有长期记忆 ✅
- 都能生成课件/笔记 ✅
- 都支持本地/自托管 ✅
- **但都是"辅导模式"（帮学生学已有的东西），不是"problem-posing 模式"（让学生自己提出问题并驱动学习）**
- **都没有 Min 的认知科学/教育研究根基**

### ⭐ 我们的差异化（基于竞品分析）

| 维度 | Khanmigo | 开源 Tutors | **Pose & Learn（我们）** |
|------|---------|------------|----------------------|
| **学习动机来源** | 外部（做题/完成作业）| 外部（理解上传的材料）| **内部（学生自己想学的东西）** |
| **AI 角色** | Socratic 审问者 | 辅导员/材料解释者 | **学习伙伴 + 镜像** |
| **核心交互** | AI 问 → 学生答 | 学生问 → AI 答 | **双向：AI 验证 + 学生提出** |
| **有没有课件** | 无（纯对话）| 有（基于材料生成）| **有（AI 生成可视化课件）** |
| **可视化** | 无 | 弱 | **强（Flipbook 式概念可视化）** |
| **评判机制** | 有（做对/做错）| 有（quiz 分数）| **无（镜像，不打分）** |
| **长期记忆** | 弱 | 有（Memory Graph）| **有（学习记录 + 知识架构）** |
| **理论基础** | Socratic 方法（执行差）| 技术驱动 | **Problem-posing + 元认知科学（Min 的 PhD）** |
| **目标用户** | K-12 做作业的学生 | 上传材料的自主学习者 | **自主学习者（含成人），学生主导** |

### Flipbook 式可视化的启发

Min 提到的 Flipbook 概念 —— 提一个东西，生成插画式详解（原理、结构、图片、场景细节）—— 补足了 Teach skill 缺少可视化的短板。

**产品可以结合的方向：**
- 课件不只是文字，而是**图文一体的概念卡**
- 学"光合作用" → AI 生成结构图 + 分步动画 + 概念关系图
- 学"法国大革命" → 时间线 + 人物关系图 + 地图
- 这部分可以参考/集成开源可视化工具（Mermaid diagrams、Excalidraw、或 AI 图像生成）

### 💡 竞品分析的核心结论

> **Khanmigo 的失败 = 我们的机会。** 它失败不是因为 Socratic 方法不对，是因为它**把方法套在了错误的学习动机上**（学生做作业 vs 学生想学）。
>
> 如果学习是**学生自主发起的**（"我想学这个"），那 Socratic 对话 + problem-posing 就能真正生效 —— 因为学生的动机是理解，不是完成任务。
>
> **"学生主导的自主学习"是让 problem-posing 真正能成功的前提条件。** 这是 Khanmigo 没搞对的，也是我们的切入点。

## 14. ⚡⚡ 最终定位：不是 Tutor，是"思维脉络生成器"

> 2026-07-11 —— 最重要的一次定位澄清

### Min 的核心定义

**"这不是 tutor。Tutor 是辅导你学习的。这个是 —— 我完全掌控自己怎么学习。"**

> "你想象成一个课本，但它完全根据你的思维脉络来教你。最终会生成一个完全根据你思维脉络而写的、关于某个话题的课文。"

### Tutor vs 这个产品的本质区别

| | Tutor（辅导员） | **这个产品** |
|---|---|---|
| **谁定义学什么** | 外部（课纲/作业/考试）| **你自己** |
| **学习路径** | 预设的（按教材走）| **你的思维脉络实时生成** |
| **终点产物** | 你"学会了"（抽象的）| **一个属于你的知识 artifact（课文/文档/稿子）** |
| **AI 的角色** | 辅助你到达预设终点 | **跟随并结构化你的思考** |
| **类比** | 教练带你练达标 | **一个会思考的笔记本，长成了你的课本** |

### ⭐ 终极用例：Content Creator 的深度学习工作流

Min 描绘了一个非常有说服力的场景：

```
你是一个 content creator，想做一个话题的视频（比如 40 分钟深度讲解）

传统做法：
  直接问 AI "帮我写一个关于 X 的稿子"
  → AI 生成一篇通用内容
  → 发出去可能不激起任何水花
  → 跟你没有真实关系

我们的做法：
  你用这个产品，从"我想学 X"开始
  → AI 跟随你的思维脉络，一节一节地生成个性化课件
  → 你提问、扩展、走岔路、回来
  → 最终积累成一个完整的、多篇章的知识文档
  → 这个文档是你真实思考过程的产物
  → 你把它提炼成口播稿 → 做成视频
  
  结果：内容有含金量，因为它是你的思维过程，不是 AI 随便创作的
```

### 这个定位为什么比"Tutor"好 10 倍

1. **有明确的终点产物** —— 不是"你学会了"（抽象），而是"你产出了一个东西"（具体）
2. **有明确的用户场景** —— content creator / 知识工作者 / 终身学习者
3. **产物有独特价值** —— 因为编码了你的思维路径，不可复制，不是 AI slop
4. **学习是副产品，产物是主产品** —— 降低"学习"的心理门槛（很多人怕"学习"，但不怕"做一个东西"）
5. **解决 AI 内容泛滥的问题** —— 市面上 AI 生成的垃圾内容太多，"经过深度思考的、有个人印记的内容"反而稀缺

### 产品名称方向（待定）

不再用 "Pose & Learn"（太像学习工具）。可能的方向：
- 某种"思考伴侣/思维脉络"的概念
- 强调"产出"而非"学习"
- Min 可以再想想

### 对技术架构的影响

这个定位意味着产品需要：
1. **课件生成能力**（图文一体的 HTML/文档，不是纯聊天）
2. **长期记忆**（跨 session 记住你的思维脉络）
3. **多篇章组织**（一个话题可能产出很多节课件，需要有目录/结构）
4. **导出能力**（最终能导出成文档、稿子、甚至视频脚本）
5. **思维脉络可视化**（让用户看到自己的学习路径像一棵树一样长出来）

### 🔵 DeepTutor（HKUDS）— 25.5K ⭐，重量级

**规模：** 25.5K stars，3.5K forks，1021 commits，v1.5.1（3天前）。这是目前最火的开源 AI 教育项目。

**核心定位：** "Agent-native learning workspace" —— 把辅导、解题、quiz、研究、可视化、mastery practice 全部放在一个可扩展系统里。

## 16. ⚠️ 与 DeepTutor 的重合度辨析 + 真正的差异化

> **"你可以把你的整个学习过程呈现出来，展现给别人看。"**

| 维度 | DeepTutor | **我们** |
|------|-----------|---------|
| **学习过程的终点** | 你"掌握了"（私有）| **一个可展示的 artifact（公开）** |
| **学习过程本身的价值** | 仅为学习者服务 | **学习过程本身是内容/产品** |
| **产物形式** | 内部笔记 | **视频稿、文档、可分享内容** |
| **受众** | 只有学习者自己 | **学习者 + 其他人（观众/读者）** |

### 🎬 关键场景：学习过程 → 内容产出管线

Min 描绘的产品不只是"学习工具"，而是一个**从学习到内容产出的完整管线**：

```
学习过程（输入端）                    内容产出（输出端）
┌──────────────────┐                ┌──────────────────────┐
│ "我想学 X"         │                │ 完整知识文档          │
│ ↓ AI 跟随思维脉络  │   ──────→     │ ↓ 提炼                │
│ ↓ 一节一节课件     │                │ 口播视频稿            │
│ ↓ 提问/扩展/回来   │                │ ↓ 可接入其他 Agent     │
│ ↓ 知识图谱长出来   │                │ 视频剪辑 / 发布        │
└──────────────────┘                └──────────────────────┘
```

**核心洞察：学习过程本身是有价值的内容。** 不是"学完了去做内容"，而是"学习过程的可视化记录 = 内容本身"。

### 🔌 Agent 接口 / 管线化设计

Min 提出的另一个前瞻性想法 —— 产出端可以**接入其他 AI agent**：

```
你的学习文档（结构化知识 artifact）
  ↓ 接口
视频制作 Agent → 读文档 → 生成视频脚本 → 剪辑
  ↓ 接口
内容分发 Agent → 多平台发布
  ↓ 接口
社区互动 Agent → 回答评论中的延伸问题
```

这意味着产品的产出不是终点，而是**管线的起点**。学习文档成为可被其他 agent 消费的标准化输入。

### 📐 修正后的定位

结合以上，产品定位应该进一步精确为：

> ~~"自适应学习工具"~~ （太像 DeepTutor）
> ~~"思维脉络生成器"~~ （只说了输入端）
>
> **"一个把你的学习过程转化为可展示知识产物的系统"**
> - 输入端：problem-posing 驱动的自主学习
> - 输出端：结构化的、可分享的、可接入其他 agent 的知识 artifact

### 这个定位为什么能站住

1. **DeepTutor 不做产出** —— 它的目标是让你学会，不是让你产出。这是清晰的品类区隔。
2. **Khanmigo 不做产出** —— 纯对话，无课件，无 artifact。
3. **OpenTutor 不做产出** —— 基于 uploaded 材料，无"思维脉络 → 内容"管线。
4. **市面上的 AI 内容工具（Jasper、Copy.ai 等）不做学习** —— 直接生成，无深度思考过程。
5. **Min 的 problem-posing 研究是两者的桥梁** —— 它既是学习方法（输入端），又是内容质量的保证（输出端有含金量）。

### 一句话差异化陈述（elevator pitch 方向）

> "DeepTutor 帮你学会一个话题。我们帮你把学习一个话题的**过程**，变成一篇有含金量的内容。"

或：

> "别人用 AI 直接生成内容（没灵魂）。我们用 AI 陪你深度学习一个话题，然后把你的思考过程变成可发布的内容（有灵魂）。"

## 17. ✅ 最终定位决策：内容创作者工具（学习是手段）

> 2026-07-11 —— 战略聚焦

### Min 的决策

> "一个产品的用户和定位应该相对准确。如果是两个方向（学习 OR 内容创作），别人不知道这是什么工具。既然 DeepTutor 已经做了'学习某个知识'，我应该更侧重内容创作方向。"

**核心逻辑：**
1. "既能学习又能创作" = 定位模糊 = 谁都抓不住
2. DeepTutor 已经占了"学一个话题"的心智位置
3. 内容创作市场更大、用户付费意愿更强、传播能力更强
4. 差异化不在于"你产不产出内容"，而在于**"你的内容为什么跟别人不一样"** → 因为它编码了你的思考方式

### 🎯 锁定定位

> **这是一个内容创作者工具。** 学习是手段，产出有思考含金量的内容是目的。

### 内容差异化的核心论点

Min 提出的关键洞察 —— 为什么"你"应该产出内容：

> "你之所以觉得你应该产出内容，是因为你的思考方式跟别人不一样，你解一个问题的方法跟别人不一样。"

这就是与 Jasper / Copy.ai / 直接问 ChatGPT 的根本区别：

| | 传统 AI 内容工具 | **这个产品** |
|---|---|---|
| 内容从哪来 | AI 直接生成 | **AI 辅助捕捉你的真实思考过程** |
| 内容的灵魂 | 没有（AI slop）| **有（编码了你的认知路径）** |
| 你的角色 | 点一下"生成" | **主导整个思考过程** |
| 结果可复制性 | 任何人都能生成一样的 | **不可复制（是你的思维指纹）** |

### 📐 两个用户场景，共享同一个管线

虽然聚焦内容创作，但产品天然支持两种场景 —— **关键是它们走的是同一个管线**，不需要做两个产品：

```
共享管线："我想学 X" → AI 跟随思维脉络 → 课件/笔记/知识结构积累
                                    ↓
                    ┌───────────────┴───────────────┐
                    ↓                               ↓
            场景 A：内容创作者                场景 B：自我学习者
            产出：视频稿、深度文章              产出：思维导图、插画式总结
            目标：发布给受众                   目标：自己的学习 artifact
            （公开）                          （私有）
```

**Min 对场景 B 的描述：**
> "生成一个思维导图，生成一个很可爱、像插画形式的，总结你都学了什么。这样你回看这个图片或历史时间地图，就能知道你这课学了什么。这比罗列的大纲有意思多了。"

### 场景 B 的"可爱插画式总结"的价值

这个细节很重要 —— 它不是一个"学习成果报告"，而是一个**有审美价值的、你自己愿意回看和分享的 artifact**：
- 思维导图（知识结构可视化）
- 插画式概念卡（像 Flipbook 那种）
- 历史时间线 + 地图
- 比干巴巴的大纲/笔记有意思得多

**这本身也是内容** —— 很多学习类 KOL 就是靠分享自己的精美学习笔记/思维导图火的。

### 对 MVP 范围的影响

聚焦内容创作后，MVP 应该优先做什么：

| 优先级 | 功能 | 为什么 |
|--------|------|--------|
| P0 | 思维脉络驱动的学习对话（problem-posing 核心）| 这是整个产品的灵魂，没有它就不是"有思考的内容" |
| P0 | 学习过程的可视化 artifact（思维导图/概念卡）| 最小可分享产物，证明"思考含金量" |
| P1 | 导出能力（Markdown / PDF / 图片）| 让 artifact 能被拿走/分享 |
| P1 | 视频稿生成（从学习文档提炼口播稿）| 内容创作者的核心需求 |
| P2 | Agent 接口（接入剪辑/发布 agent）| 管线化的前瞻功能 |

### 更新后的 elevator pitch

> **"AI 能帮你写内容，但写出来的东西没有你的灵魂。这个工具陪你深度思考一个话题，然后把你的思考过程变成别人愿意看的内容。"**

### 产品名方向（更新）

聚焦内容创作 + 思考过程，名字应该传达"思考 → 产出"：
- 不再是 "Pose & Learn"（学习感太重）
- 不再是 "思维脉络生成器"（太抽象）
- 方向：强调"你的思考变成了看得见的东西"

## 19. 🔄 双模式设计：Explore Mode & Create Mode

> 2026-07-11 —— Min 提出的核心产品架构

### 两种用户状态

用户来到这个工具时，可能处于两种完全不同的状态：

**模式 A：Explore Mode（探索模式）**
> "我对 X 感兴趣，但不太清楚它到底是什么，想先学学看。"

- 用户只有一个模糊的话题（"我想了解量子计算"、"法国大革命是怎么回事"）
- 没有明确的产出目标
- AI 的角色：跟随用户的好奇心，一节一节生成课件，用 problem-posing 引导深度思考
- 产出：知识结构、思维导图、概念卡（给自己看的 learning artifact）

**模式 B：Create Mode（创作模式）**
> "我想做一个关于 X 的内容，我大概知道方向，需要把它做出来。"

- 用户有明确的产出意图（"我要做一期关于 AI 安全的视频"、"我要写一篇深度文章"）
- 可能已经有一些知识基础，也可能需要边学边做
- AI 的角色：帮用户补充知识盲区 + 把思考过程结构化成可发布的产物
- 产出：视频稿、深度文章、口播脚本（给别人看的内容产物）

### ⭐ 关键：模式之间可以无缝切换

> "你学了一半想把某个小模块写成发表的东西也是可以的。"

```
Explore Mode                          Create Mode
┌─────────────────┐                  ┌─────────────────┐
│ 学习"法国大革命"  │                  │ 做"AI安全"视频   │
│ ├ 第1节: 起因    │   ──切换──→      │ ├ 补充知识盲区    │
│ ├ 第2节: 过程    │                  │ ├ 结构化论点     │
│ ├ 第3节: 影响    │                  │ ├ 生成稿子       │
│ │  └ 💡"这段很好  │   ←─切换──       │ └ 导出           │
│ │     写成一篇"   │                  │                 │
│ └ 第4节: ...     │                  └─────────────────┘
└─────────────────┘
```

**场景举例：**
- 用户在 Explore Mode 学"法国大革命"，学到第 3 节"人权宣言的影响"时觉得这段特别好 → 切换到 Create Mode → 把这个模块提炼成一篇可发布的文章
- 用户在 Create Mode 做"AI 安全"视频，发现对"alignment problem"的理解不够深 → 切换到 Explore Mode 深入学习 → 学完再回来继续写稿子

### 为什么必须分模式而不是混在一起

| | 混在一起 | 分模式 |
|---|---|---|
| **用户心智** | "这是学习工具还是创作工具？" | 进入时选择，预期清晰 |
| **AI 行为** | 不知道该继续教还是该帮你写 | 模式明确，AI 行为精准 |
| **UI 焦点** | 课件和创作工具挤在一起打架 | Explore = 课件优先；Create = 编辑器优先 |
| **产物期望** | 模糊 | Explore = learning artifact；Create = publishable content |

### 技术影响

- 两个模式共享同一个底层管线（思维脉络记忆 + problem-posing 引擎 + 知识结构）
- 区别在于**前端的呈现重点**和**AI 的行为模式**：
  - Explore Mode：课件展示优先，对话辅助
  - Create Mode：编辑器优先，知识补充和结构化辅助
- 切换时不需要丢失 context —— 学习积累的知识结构在两个模式间共享

- [ ] **Problem-posing 评估的其他学术框架**：除了 complexity scoring，还有哪些界定"好问题"的方式？（Silver 的 originality 框架、Leung 的 creativity 维度、solvability/feasibility 等）→ 需要文献综述

## 20. 🎯 目标用户画像 + 商业模型

> 2026-07-13 —— 目标用户收窄至个人 IP 内容创作者

### 不是谁

- ❌ 企业内容团队（多用户/classroom）
- ❌ K-12 学生做作业
- ❌ 纯粹想"学会一个东西"的被动学习者

### 目标用户

**个人 IP 内容创作者** — 一个人（或极小团队）经营自己的知识品牌：

| 类型 | 典型场景 | 产出形式 |
|------|---------|---------|
| **YouTube 知识类博主** | 做深度视频（5-40min），需要先真正搞懂一个话题 | 视频脚本、分镜稿 |
| **Newsletter 写手** | 写深度邮件通讯（Substack 等），需要持续产出高质量内容 | 长文、系列专栏 |
| **博客作者** | 写深度博客文章，建立个人专业品牌 | Blog post、系列文章 |
| **播客/口播创作者** | 需要把复杂话题讲清楚、讲有趣 | 口播稿、提纲 |

### 用户的核心痛点

> **"我想做一个关于 X 的深度内容，但我对这个话题的理解不够深，直接问 AI 写出来的东西没有灵魂。"**

现有选项：
- 自己花大量时间查资料 → 慢，且容易碎片化
- 直接让 AI 写稿 → 快，但没有个人思考痕迹，是 AI slop
- 看视频/读书学习 → 学到了，但没有结构化的产出管线

**我们的位置：学习 → 产出之间缺失的那座桥。**

### 商业模型（初步）

**阶段 1（MVP / Portfolio）：** 免费开源，个人网站展示用。零商业压力，验证产品假设。

**阶段 2（如果验证成功）：**
- **Freemium**：基础功能免费（话题探索 + 问题驱动学习），高级功能付费（多格式导出、风格模板、AI 海报生成、多 LLM 支持）
- 或者 **Pro 订阅**：月费制，解锁所有产出模板和高级可视化
- 核心逻辑：**创作者的付费意愿 > 学生**，因为产出直接关系到他们的收入和品牌

**不做的：**
- ❌ 不做广告
- ❌ 不卖数据
- ❌ 不做 enterprise SaaS

```
AI 响应：开启欧洲专题 Track B（E0-E5，四轨并行：西欧/中欧/东欧/北欧）
  → 不再是单一线性课程，变成了双轨并行
  ↓
0005 帝国时代
  ↓ 💬 Min 反馈："一战太薄了，缺参战国、导火索、奥斯曼为什么站德国"
  ↓
AI 响应：开附录 0005b（一战专课）+ 补充参考文档
  ↓ 💬 Min 提问："为什么这几国能殖民？"
  ↓
AI 响应：在 0005 中补"为何列强能殖民"三节表，改下一课
```

### 思维脉络的特征（从实例中提炼）

**1. 不是线性知识树，是问题驱动的分支结构**

```
        0001
          ↓
        0002 ──→ 0002b(E0) ──→ 欧洲专题 Track B
          ↓                        ├ E1 封建教会
        0003                        ├ E2 英法分家
          ↓                        ├ E3 罗斯/俄乌
        0004 ──→ 💬"欧洲内部?"      ├ E4 地中海商业
          ↓                        └ E5 大航海 → 汇合回 0005
        0005 ──→ 💬"一战太薄" → 0005b(附录)
          ↓
        0006
```

- `prerequisite` 边：0001→0002→0003（课程的前置依赖）
- `question-spawned` 边：0004→Track B（Min 的问题引发的分支）
- `appendix` 边：0005→0005b（反馈引发的补充）

**2. 每个分支节点是一个真实的问题或反馈**
- 不是 AI 预设的，是用户自己提出的
- 问题本身是元数据，记录在 learning record 里

**3. 知识结构是活的、被雕刻过的**
- 最终的课程结构 ≠ 任何一本教科书
- 它是 Min 的好奇心雕刻出来的独特路径
- 四轨并行地图（西欧/中欧/东欧/北欧）就是例子 — 这不是按年份流水账，是按"谁从哪来"的角度

**4. 最终产物是一个可浏览的知识网站**
- HTML 课程 + 参考文档 + 学习记录
- 部署在 Vercel 上：world-history-fawn.vercel.app
- 别人看到的是"一个完整的、有结构的知识库"

### 技术建模：Question-Annotated Topic Graph

```
TopicGraph {
  nodes: KnowledgeModule[]     // 每个课件/知识模块
  edges: Edge[]                // 连接关系

  Edge.type:
    - "prerequisite"           // A 必须在 B 之前学
    - "question-spawned"       // 用户的问题引发了 B（分支）
    - "appendix"              // B 是 A 的补充/附录
    - "merge"                  // 两条轨道汇合

  Edge.metadata:
    - question: string        // 什么问题触发了这条边？
    - timestamp: Date
    - user_note: string       // 用户的原话/反馈

  KnowledgeModule:
    - id, title, content
    - type: "lesson" | "reference" | "appendix" | "review"
    - status: "completed" | "in-progress" | "planned"
    - highlights: string[]    // 用户的 aha moments
    - questions: string[]     // 用户在这个模块中提出的问题
}
```

### 可视化

- **知识网络图**：节点 = 课件，边 = 关系，颜色区分类型（prerequisite / question-spawned / appendix）
- **问题标记**：在边上标注触发问题，鼠标悬停可以看到"Min 问了什么才有了这个分支"
- **进度覆盖**：已学模块亮起，待学灰暗
- **不是时间线，不是树，是网** — 因为问题会让知识结构长出意想不到的连接

### 产出格式的细节

**Highlight 卡片（Aha Moments）：**
- 自动从学习对话中提取用户的"洞察时刻"
- 例如："十字军东征最终削弱了自己这边"、"伊斯兰分裂的根源不是神学而是政治"
- 可以单独作为社交媒体卡片分享

**智能格式建议：**
- AI 评估当前探索深度 → 建议最合适的产出格式
- 短视频（1min）：< 3 个 highlight，适合短视频平台
- Blog post：5+ 个 highlight，有完整的知识脉络
- 长视频（10min+）：需要深入探索 + 多个分支 + 附录
- 如果内容量不够长视频 → 建议先做短视频，并指出"继续探索哪个方向可以扩写"

**外部 Skill 集成（可选，MVP 后）：**
- 风格化海报：可连接 baoyu-infographic / comfyui 等技能
- 视频生成：可连接外部 agent
- MVP 阶段不需要，但架构上预留接口

### openflipbook 概览

**仓库：** `eren23/openflipbook`（172⭐, MIT, 活跃维护）
**定位：** flipbook.page 的开源 clone —— "image-is-the-UI" infinite canvas

**核心循环：** 输入话题 → AI 生成插画式页面 → 点击任意区域 → 视觉模型解析点击 → 生成下一页 → 无限探索

### 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| **前端** | Next.js 15 + TypeScript | 跟 Min 已会的栈吻合 |
| **后端** | FastAPI (Python) + Modal | Modal = serverless Python，按调用计费 |
| **图片生成** | fal nano-banana-pro | 插画风格，$0.02-0.04/页 |
| **视觉理解** | Gemini 3 Flash (via OpenRouter) | 解析点击位置 → 生成下一页 prompt |
| **动画** | fal-ai/ltx-video | 图片→5s MP4，$0.02/clip |
| **存储** | Cloudflare R2 + MongoDB | 图片 + node graph |
| **部署** | Docker (本地) 或 Modal+R2 (云端) | `make demo` 一键启动 |

### openflipbook 的能力与 explore-craft 的映射

| openflipbook 能力 | explore-craft 怎么用 |

| **一页一插画** | 每个知识模块 → 一页精美的概念插画 |

| **时间轴 film-strip** | 思维脉络可视化 —— 看到你的探索路径 |
| **风格固定（📌 pin style）** | 一个学习 session 统一视觉风格 |

| **动画 toggle** | 短视频产出（highlight → 5s 动画） |
| **World Mode（空间导航）** | 知识网络图变成立体空间，"走进"一个话题 |
| **Critic loop（自动质检）** | 生成的页面有质量保证 |
