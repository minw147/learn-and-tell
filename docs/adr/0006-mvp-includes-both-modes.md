# MVP scope includes both Explore Mode and Create Mode

The MVP was initially scoped as Explore Mode only, deferring Create Mode to a later milestone to keep the first build small. That call was reversed within the same planning session: the learning-to-output pipeline is the product's core differentiator (vs. DeepTutor/Khanmigo, which stop at "you learned it"). Shipping Explore-only would validate a generic tutoring loop and leave the actual pitch — publishable content with a real thinking-path behind it — unverified.

Scope should instead be cut from secondary features: interactive frontend visualization, multi-session memory, and openflipbook integration all remain deferred to Phase 2. Recorded because a future reader will find "Explore + Create in MVP" surprising given how much bigger it makes the first build, and might try to cut Create Mode again without knowing it was already considered and rejected once.
