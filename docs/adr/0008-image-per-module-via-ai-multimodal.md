# Every module gets a generated concept illustration

Each module, when generated during Explore Mode, is paired with one AI-generated concept illustration (via the `ai-multimodal` skill / Gemini image generation), stored in `./images/` and bound to the module. This happens even for non-highlight modules — not gated on highlight status. Create Mode reuses these images or regenerates them at higher "publish quality" for the final artifact.

Considered gating image generation to highlights only, which would cost less but leave most of Explore Mode as plain Tufte-style text — undermining the "image-is-the-UI" experience borrowed from openflipbook, which was the whole point of adding images at all. This still counts as "no frontend": the module remains a static HTML file with embedded images, not a running app. It does introduce a hard dependency on `ai-multimodal`/Gemini and a per-module generation cost, which is why it's recorded — a future reader optimizing for cost might reasonably try to cut this back to highlights-only without knowing the fuller version was chosen deliberately for the Phase 2 handoff (accumulated `./images/` becomes migratable data for the openflipbook fork).
