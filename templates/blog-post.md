# Blog Post Template

For a connected arc across several highlights — see `references/OUTPUT-TEMPLATES-GUIDE.md` for fit judgment.

```md
# {Title}

*{One-line dek/subtitle — states the reframe, not just the topic}*

## Opening
{Start from the question that drove the exploration, not a topic
definition. "I assumed X, until I discovered Y" style openings work well when the
user's own problem-posing produced a real reversal.}

## {Section per highlight or branch}
{Each section should map to one confirmed highlight or one graph branch.
Use the module's citations from RESOURCES.md — carry real links into the
post, don't drop them. Embed the module's ./images/ illustration if it
fits the section, or note where a higher-quality regeneration is needed.}

## {Connective section, if multiple branches converge}
{Where the piece pulls separate branches together — this is often where
the strongest content lives, since it's insight the source material
doesn't state directly.}

## Closing
{Land on the highlight that best summarizes the arc. Optionally point to
what's still unexplored — an honest "here's where I'd go next" beats a
forced neat bow.}

---
**Sources**: {carry forward the RESOURCES.md citations actually used}
```

## Rules

- Minimum viable: 2+ confirmed highlights that connect into an arc. A single highlight is a short-video, not a blog post — say so if that's what the material actually supports.
- Preserve the user's own phrasing from their problem-posing questions where it's sharp — that's the "thinking fingerprint" the whole product is built around, not something to smooth over into generic prose.
- Causal/necessity framing over chronological listing, per the user's observed narrative preference, unless `learning-style.md` says otherwise for this user.
