# Long Video Script Template

For 10min+ deep-dive content — needs real narrative depth: multiple branches, appendices, or a track explored across several sessions. See `references/OUTPUT-TEMPLATES-GUIDE.md`; if the graph doesn't have this much branch coverage yet, offer to dig deeper or broaden scope before drafting this template.

```md
# {Title}

**Target length**: {10min / 20min / 40min}
**Branches drawn on**: {list the TOPIC-GRAPH.json branches/nodes this pulls from}

## Cold open
{The single most compelling hook from the strongest highlight — same
principle as the short-video hook, but here it's a promise for a longer
payoff, not the payoff itself.}

## Act 1 — Setup
{Establish the main-track context needed to make later branches land.
This is prerequisite-edge territory: what does the viewer need before
the interesting parts make sense?}

## Act 2 — {One section per major branch}
{Each act section corresponds to a question-spawned or appendix branch
in the graph. Structure it as: what question opened this branch → what
was found → how it reconnects to the main thread (merge edge, if any).}

## Act 3 — Synthesis
{Where branches converge. This is usually the section with the highest
highlight density — the connections that only exist because the
exploration went non-linear.}

## Close
{Return to the cold open's promise and resolve it. End on the sharpest
highlight, in close-to-original phrasing.}

## Visual/pacing notes
{Map ./images/ assets to acts; flag which need publish-quality
regeneration. Note where pacing needs a breather vs where it should
accelerate (mirrors the graph's no-question streaks — sections the user
moved through fast can usually move fast on screen too).}
```

## Rules

- Requires multiple branches — a single linear run of modules, however long, is a blog post or a short video, not this template. Don't stretch thin material to fill runtime.
- Preserve branch structure in the outline — viewers should be able to feel that this wasn't a straight-line script, it followed real curiosity.
